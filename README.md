# Sales & Customer Analytics Project


# Overview:
Designed and implemented an end-to-end SQL-based data analytics project using a structured dataset containing sales, customers, and products data. Focused on extracting meaningful business insights and building a strong analytical foundation similar to industry-level reporting systems.

# Key Contributions & Learnings:

# Data Modeling & Joins
Worked with multiple tables (fact & dimension tables)
Implemented efficient joins to combine customer, product, and sales data
Avoided common pitfalls like Cartesian joins and optimized query performance

# Sales Performance Analysis
Analyzed total sales by product, category, and time (monthly/yearly trends)
Identified top-performing products and revenue drivers
Built queries for trend analysis and growth tracking

# Customer Analysis
Calculated key metrics:
Total spending per customer
First and last purchase date
Customer lifespan (using date functions)
Derived insights on customer behavior and engagement

# Data Segmentation
Performed category-level sales aggregation
Calculated percentage contribution of each category using window functions
Built foundation for ABC segmentation (high-value vs low-value categories)

# Advanced SQL Techniques
# Used CTEs (Common Table Expressions) for modular and readable queries
# Applied Window Functions (SUM() OVER(), RANK()) for advanced analysis
# Implemented date functions for time-based insights
Practiced query optimization and debugging techniques

# Error Handling & Debugging
Resolved real-world SQL issues such as:
Incorrect joins leading to performance issues
Function compatibility across SQL dialects
Query optimization for large datasets

# Business-Oriented Thinking
Converted raw data into business questions:
Which products drive the most revenue?
What is customer lifetime value?
How does sales change over time?
Focused on insights that are directly useful for decision-making
